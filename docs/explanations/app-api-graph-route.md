# app/api/graph/route.ts

## このファイルは何をするのか
ナレッジグラフ表示用のノード・エッジデータを構築するAPIエンドポイント。

## なぜこのファイルが必要なのか
ノート・プロジェクト・リンクの関係性をグラフとして可視化するため。フロントエンドのCanvas描画に必要なデータ構造（ノードとエッジ）をサーバー側で組み立てる。

## コードの概要
- **GET**: ノート（最大100件）・プロジェクト・ノートリンクの3つを並列取得
- ノートノード: タグに応じた色分け（アイデア=黄、情報=青、ToDo=緑）
- プロジェクトノード: ピンク色で表示
- エッジ: ノートリンク（link型）、ノート→プロジェクト（project型）の2種類
- タグによるグルーピング用の構造も用意（同タグ3件以上で代表ノードに接続）

## 依存関係
- 使っているもの: `next/server`, `@/lib/supabase/server`
- 使われているもの: `app/(main)/graph/page.tsx`
