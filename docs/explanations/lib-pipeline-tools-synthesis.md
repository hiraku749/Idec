# lib/pipeline/tools/synthesis.ts

## このファイルは何をするのか
複数ノートの共通テーマ・矛盾点・シナジーを分析し、統合レポートをノートとして生成するパイプライン。

## なぜこのファイルが必要なのか
ユーザーが蓄積した複数のアイデアノートを横断的に分析し、新たな気づきや統合的な視点を得るため。個別に読むだけでは見えない関連性をAIが発見してくれる。

## コードの概要
- `runSynthesis()` は2〜5個のノートIDを受け取り、各ノートを取得してコンテキストブロックに変換
- AIに「共通テーマ」「相違点」「シナジー」「アクションアイテム」の4セクション構成でレポート作成を依頼
- 結果を `executeOutputAction` で新規ノート（タグ: アイデア）として自動保存
- ノート数のバリデーション（2〜5個）をパイプライン内で実施

- 使われているもの: `app/api/synthesis/route.ts`, `lib/pipeline/index.ts`
